Embedded Commands of the FlexVoice Engine

While you are reading a text with FlexVoice you can change all sort of parameters, like voice, pitch, speed etc. (more below in the table). It is very useful to bring a colour in the speech making it more interesting, or to highlight a part of the text. The Tags are only used by the engine, and are not read out loudly.

A simple sample text: {\Vce=Speaker="Tom"\ Hello I'm Tom. \Vce=Speaker="Julie"\ and I'm Julie.} The first reader is Tom introduces itself and "gives the microphone" to Julie.

The Available control tags and their syntax are listed below:
\Controltag\
\Controltag='##'\
\Controltag='$$'\

 Where '##' is a numeric parameter and '$$' is a string parameter. The string value could be enclosed between all sort of characters ' {} [] < > " or even it is possible not to use any of them
The fractional parameters must be set as percent.

The available tags:

Basic Commands
vce=speaker="$$"
 sets the speaker to read the text after the tag. If the supplied speaker is a non-existing one on the system simply neglects this tag.
vol=##
 sets the volume the reading.
spd=## Sets the default speed of the reading in Word Per Minute. The speed of a reading is not an exactly defined measurement!
rspd=##
 sets relative speed to the default (in percent)  
pau=##
 inserts a silence with the given length (in ms)
pit=##
 sets the default pitch (in Hz)
rpit=##
 sets the relative pitch to the default (in percent)
emp=## Tells the engine the level of accent on the following word. There are 4 levels, starting from 0 (with no emphasis at all) to the maximal level of 3.
intlevel=##
 sets the intonation level (in percent), the lower the number is, the reading is more monotone.
Pronunciation and spelling commands
pron="$$"
 Describes the pronunciation for the next block.
The block ends with \endpron\ The given phoneme
string must follow the documentation
endpron
 closes the pronunciation block
wpron="$$"
 sets the pronunciation for the next word.
wspell=## Spells out a word, instead of trying to read it out, like FBI
spell
 spells the next block. The block ends with \endspell\
endspell
 closes the spell block started by \spell\ tag.
sayas Tells the engine to start pronouncing a block in the given manner.
endsayas The end marks for the block started by the sayas
wsayas=$$ Tells the Engine to pronounce the following word in the given way
wordmode=on|off Reads every word in an accented manner, and keeps a slight break after very word. To be used, when the speech has to highlighted extremely.
lang=$$ Sets the language used for processing and reading the text after the tag. Possible entries: eng, ger and hun
endlang Sets the end boundary for the language set by Lang tag.
wlang=$$ Sets the language used for processing and reading of the following word see Lang
langdet on|off turns the automatic language detection on or off.
aspell Tells the engine to spell out words letter-by-letter. Like C-H-A-I-R
endaspell Closes the block started by aspell
waspell Tells the engine to spell out the following word letter-by-letter
Dictionary handling commands
loaddict "$$","$$"
 Loads a user dictionary file with a specified name.
unloaddict "$$"
 Unloads a dictionary file with a specified name.
enabledict "$$"
 Enables a dictionary.
disabledict "$$"
 Disables a dictionary.
pwd=##
 password
adddict Adds a new dictionary.
removedicts Removes all dictionaries.
enabledicts Enables the dictionaries.
disabledicts Disables all dictionaries.
removedict Removes a dictionary.
Advanced commands
eng *
phstr=## ** Tells the Engine, that the input is a phoneme string.
mrk=## places a user marker, for the engine to issue a bookmark event for the programmer.

*The control tag Eng has special syntax:
\Eng:%%\ or \Eng;{CLSID}:%%\
where `CLSID' is the CLS ID of the engine and `%%' is one of the following:

[:phone arpa TruVoice][@@]
[:phone arpa Sampa][@@]
[:phone arpa DEC][@@]

`@@' is a phoneme sequence with syntax $$<##,##>
where `$$' is the phoneme symbol,
the first `##' is the duration in milliseconds,
the second `##' is the pitch of the phoneme (in the range 0-31 it is in relative semitones, from 32 it is in Herz).
Phonemes should be written without any separator.

**Phoneme strings for FlexVoice Engine.
If you would like to pronounce a special word, or to make the engine sing or to modify the pronouncing of a word, it is wise to use phstr embedded command.

Usage

phoneme_code: < duration, pitch-contour[,loudness-contour] >


Where
duration: in milliseconds, This is the duration of the phoneme.

pitch-contour:
Syntax: pitch_value:position
Where
pitch_value is the value of the pitch in Hz
position is the position where the pitch value should take its full value in % of the full duration of the phoneme

Describing the pitch contour of the phoneme. By default the pitch supplied here is valid in the middle of the duration (of course if it's not defined in a different way) The engine automatically creates the transition between the prevoius and the following phoneme's pitch level. There is a possibility to exactly determine the value of the pitch for every interval of the duration. Like 120:30 means the pitch value of 120 Hz should be taken when the phoneme is at the 30% of the duration. Multiple pitch and "time" values are accepted by the phoneme descriptor.

loudness-contour: optional, describes the loudness boundaries of the phoneme, the numerical values should measured and compared with the one used by the engine. It is very dangereous to change this value.

For example:
Phoneme: "ju:" (from Use)
ju:<100,120>
- duration = 100
- pitch = 120 at 0 position
Here the phoneme is 0.1sec long, and has a pitch of 100Hz. The basic description of a phoneme. The pitch is set to be valid at the beginning of the phoneme.

ju:<100,120,200000>
- duration = 100
- pitch = 120 at 0 position
- loudness = 200000 (absolute loudness * 100) at 50% of the phoneme
You can add loudness to a phoneme. Elevating the voice in a word makes the remark during the speech.

ju:<100,120:30,200000>
- pitch = 120 at 30%
The pitch could be set to change at other point than the begetting. Here it is at 30% of the phoneme.

ju:<100,<120:30,150:70,200:90>,200000>
It is possible to make a description of the phoneme pronunciation. Start at the default pitch until 30% of the phoneme, than change the pitch to 120 up till 70%. At this point elevate the pitch to 150 and after 90% raise it to 200 until the phoneme is finished.

ju:<100,<120:30,150:70,200:90>,<200000:70,300000:80>>
Like in the previous example the loudness could change inside a phoneme either.
