Tutorial FVFarmServerTest

Demonstrating the power of FlexVoice Farm Server
The FvFarmServerTest application is a combined Client/Server application. It is used for testing and demostrating the FlexVocie Client/Server operation.

Usage:

FVFarmServerTest.exe [options]

Options:
I - input text (default is "Hello World").
O - output file name (default is "sample.wav).
MU - use MU-LAW encoding instead of PCM.
VOX - use VOX encoding instead of PCM.

For example: 

	FVFarmServerTest.exe
	FVFarmServerTest.exe I "This is a simple test sentence"
	FVFarmServerTest.exe O myfile.wav
	FVFarmServerTest.exe I "The text is here" O out.wav MU

The program starts with a server initialization: FVFarmServerInitialize( NULL, 1 );  Only one instance is initialized using handle NULL. This is a simple test program, so no handle should be supplied.

The next step is branching. Depending ont the required output file type (voc or PCM) the program can processes the output in a different synthax, but they are basically same, so only one is explained. 

	FILE * file = fopen( outputfile, "wb" );
            FVFarmServerSpeak( inputtext, dformat, SAMPLE_FREQ, BIT_RESOLUTION, put_VOX, (void*)file );
		    FVFarmServerWait();
            fclose( file );

A file is opened for writing(outputfile). The inputtext is saved using the dfromat SAMPLE_FREQ BIT_RESOLUTION file properties with put_VOX function (fwrite) into the file named file.

While the processing and the writing is in progress the program puts in a wait: FVFarmServerWait() 

The last step is to close everything: FVFarmServerUninitialize
