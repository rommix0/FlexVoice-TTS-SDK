The Engine

A TTS Engine is used to produce speech from text.

Creation

It is created by the Factory's createEngine method. The engine creation requires an OutputSite that tells the engine what kind of output is to generate (e.g. wave or phoneme-string) and where to put it. (see Output Sites). It also requires a default speaker and a language. The factory model ensures that the engine-creation is a relative cheap process

Speech requests

The engine can process requests (see speakRequest) that are queued and processed continuously. The input can also be given in fragments by using the addFragment method. Between the fragments the user can place own bookmarks by the addBookmark method. The speech can be stopped, paused and resumed (see stop, pause, resume). There are also possibility for skipping parts of the text (e.g. the next two sentence) online by calling the skip method. Calling the wait method it will be ensured that all the given speak requests were finished.

Speaker handling

The speakers can also be modified online and offline by using the setSpeaker method. The engine stores an inner speaker structure that is used for getting the actual speech parameters. The outer given speaker is copied to this inner instance and will change the parameters of the output speech. Setting the speaker offline (when not speaking) the parameters change immediately and the next request starts speaking by the newly set speaker parameters. In case of setting the speaker online (when speaking) the parameters will change only after few seconds. The inner speake instance can be get by calling the getSpeaker method.

The engine also stores a speaker map that lets the engine choose the speaker from in case of an embedded speaker change (e.g. \vce=speaker="Tom"\). This inner speaker map can be handled by the following methods: setSpeakers, addSpeaker, removeSpeaker, updateSpeaker, getSpeakers.
