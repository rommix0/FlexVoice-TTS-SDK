    • Tutorials Sample programs demonstrating of using FlexVoice Engine API
      • HelloWorld0 This example shows how to instantiate a FlexVoice 3.0 engine and let it speak to a file. Demonstrates the minimal set of calls to use FlexVoice 3.0 in an application.
      • HelloWorld1 This example is an extention of the previous one in as much as it finds the data directory on your system and also it loads the speaker definitions included in the voicelist.tvl file that acts as a registry of the available voices. This way, the user can change speakers using embedded commands without blocking the output of the speech.
      • HelloWorld2 In this example, a class is set up that is doing most of the work. Compared to the previous example the functionality is not extended.
      • Say This is a useful application that can be used to produce wave files using the text to speech engine. Compared to the previous HelloWorld applications it adds command line handling.

Classes

  • FakeAttribute Base must be derived from IAttribute
  • IAttribute Attribute interface Lets object hide the member variables from the client by defining parameters with their name.
  • BookmarkTypeList A BookmarkTypeList provides a comfortable interface to be used with INotifyDispatcher methods, namely with registerNotify, addBookmarkTypes and removeBookmarkTypes.
  • Bookmark Bookmarks are passed along to the IOutputSite by the Engine which forwards them to INotify instances.
  • BookmarkDocumentStructure NOT YET USED
  • BookmarkEmbedded Base class of bookmarks that are embedded into the output flow of the engine

  • Engine
  • EngineFactory Responsible for managing Engine and instances, Speaker resources and Dictionaries (DictionaryHandle).
  • IOutputFormat Abstract base class for all TTS output formats.
  • IAnnIPAPhonemeOutputSite Interface called by the TTS engine to send out annotated IPA phoneme data when it becomes available
  • IAnnPhonemeOutputSite Interface called by the TTS engine to send out annotated phoneme data (using phoneme codes) when it becomes available
  • IOutputSite Interface called by the TTS engine to send out data when it becomes available.
  • IPhonemeOutputSite Interface called by the TTS engine to send out phoneme data (in the form of phoneme codes) when it becomes available
  • IWaveOutputSite Interface called by the TTS engine to send out wave data when it becomes available.
  • GenericException Base class for all exceptions
  • WaveFileOutputSite A WaveFileOutputSite supports saving speech into a PCM formatted "WAV" file.
  • INotifier The purpose of this class is to receive and dispatch the received notifications
  • INotify Defines notification methods to be implemented by the API client
  • INotifyDispatcher Interface for managing notification dispatching.
  • Notifier An implementation of the INotifier interfacefor dispatching notifications
  • NotifyTransfer The purpose of this class is to provide an implementation of INotify such that the processing of the notifications happens on a thread different from the notification thread.
  • IPAPhoneme Describes phonemes by their features, essentially providing a language-independent representation of phonemes.
  • IPhonemeCoder Interface to convert between phoneme symbol tables and the above IPA phoneme representations.
  • FilePhonemeCoder IPhonemeCoder implementation for converting phoneme codes using phonemes described in a file
  • WavePlayOutputSite A WavePlayOutputSite supports playing waves through a hardware device.
  • WavePlayOutputSiteEx A WavePlayOutputSite supports playing waves through a hardware device.
  • Speaker Speaker class.
  • SpeakerMap
  • SpeakerMapIterator
  • ConstDictIterator Const iteration of a DictionaryHandle instance.
  • DictIterator Non-const iteration of a DictionaryHandle instance.
  • DictionaryHandle This dictionary implementation is compatible with FlexVoice 10 implementations.
  • IDictionaryItem A dictionary item accesses the word parameters.
