Introduction of the model of FlexVoice's C++ API.

First a factory (see The Factory) needs to be created that is responsible for storing the resources such as the speakers (see The Speaker), the dictionaries (see User Dictionaries) and so on. The factory is also responsible for creating the engines (see The Engine). The engine is able to create speech from text. The output speech depends on the selected speaker. The speaker is a set of parameters that specify the voice quality, prosody and so forth. During the speech, the engine sends bookmarks (see Bookmarks) to the client that notify it of the specific speech events e.g. begin, end or even the phonemes. It is possible to control where the output should go. By default the output goes to the sound card, but it can be redirected to a file for example, or the user can even take control of it and implement their own strategy to handle the output. This behaviour can be controlled by the OutputSites (see OutputSites) The user can also use user dictionaries that enable them to define the pronounciation of some specific words. For a more detailed description of the above mentioned concepts please read their specific documentation.

Common operations are:

  • create an engine factory
  • create a notification object (typically a Notifier instance) in order to receive notifications about speak-events,
  • create an output-site object to be used with the engine
  • create a new engine instance
  • pass the notification object to the output-site object,
  • send requests to the engine,
  • manipulate the engine state, in particular the properties of the speaker
  • process notification messages arriving as the engine speaks.
