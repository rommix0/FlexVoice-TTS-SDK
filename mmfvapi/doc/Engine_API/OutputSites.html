OutputSites

The OutputSites lets the user define where to put the output of the engine. it also defines the output format of the engine. The output can be either simple wave or phoneme string or even the normalized text. By default the SDK supports output sites for the most usual cases but the user can create own output sites. To do it the IOutputSite interface should be implemented. The OutputSite has to put the output data to the destination (e.g. to the sound card or to a .wav file) and it also has to deal with the dispatching of the notifications. Through its interface it gets the output data and teh notifications in streaming mode. The data should be forwarded to the destination (e.g. to the sound card o or to the .wav file) while the notifications (the bookmarks) should be passed to the listeners that was registered to the OutputSite. The IOutputSite interface itself is not able to recieve the output of the engine while it varies depending on the required output format. There are different outputs in case wave output or in case of phoneme string output. That's why the implementation shouldn't implement directly the IOutputSite interface but one of the following: IWaveOutputSite, IPhonemeOutputSite, IAnnPhonemeOutputSite, IAnnIPAOutputSite. These interfaces are inherited from the IOutputSite interface. The IOutputSite inteface has a getOutputFormat method that gives an IOutputFormat interface. This interface has the outputType method that defines the type of the OutputSite. When giving the OutputSite implementation to the engine (see factory's createEngine method) it calls this method to categorize the given OutputSite. Afgter it the engine tries to dynamic_cast the IOutputSite pointer to the one of the above mentioned specific interfaces. (note that it requires the enabling of the RTTI information in the compiler) If it suceeded the IOutputFormat will be also dynamic_cast-ed to the specific output format: WaveOutputFormat, PhonemeOutputFormat, AnnPhonemeOutputFormat. These structures contains all the information for the engine tho generate the proper output. (see also The Output Format)
