The Output Format

The engine can produce either speech (wave) or phoneme output or normalized text output (not yet implemented) .

Upon construction an IOutputSite instance is passed to the Engine (see factory's createEngine). In order to determine the type of output expected, the Engine calls the getOutputFormat method of the passed IOutputSite instance, which in turn returns an IOutputFormat instance. The engine then sets itself up so that it will be able to support the given IOutputSite instance.

The output format is dynamic_cast-ed from the IOutputFormat interface to the proper class depending on the result of outputType method.

WaveOutputFormat

The most common output is the speech. The parameters of the wave output is described by the WaveOutputFormat structure. It has the following parameters: sample frequency, bit resolution, and wave coding. For detailed description see WaveOutputFormat

PhonemeOutputFormat, AnnPhonemeOutputFormat, AnnIPAPhonemeOutputFormat

If the user wants the engine to generate phonemes as output instead of wave it has to use the IPhonemeOutputSite or the IAnnPhonemeOutputSite depending on that the prosody information is required or not. In case of special usage the instead of the phoneme codes the IPA representation can also be generated. In this case the IAnnIPAPhonemeOutputSite should be used. There's no extra information needed for the phoneme outputs, that's why the derived OutputFormat structures doesn't contain any addition.
